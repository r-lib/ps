# Send signal to a process

Send a signal to the process. Not implemented on Windows. See
[`signals()`](https://ps.r-lib.org/dev/reference/signals.md) for the
list of signals on the current platform.

## Usage

``` r
ps_send_signal(p = ps_handle(), sig)
```

## Arguments

- p:

  Process handle, or a list of process handles.

- sig:

  Signal number, see
  [`signals()`](https://ps.r-lib.org/dev/reference/signals.md).

## Details

It checks if the process is still running, before sending the signal, to
avoid signalling the wrong process, because of pid reuse.

## Examples

``` r
px <- processx::process$new("sleep", "10")
p <- ps_handle(px$get_pid())
p
#> <ps::ps_handle> PID=9000, NAME=sleep, AT=2026-03-31 12:54:00.01
ps_send_signal(p, signals()$SIGINT)
#> NULL
p
#> <ps::ps_handle> PID=9000, NAME=???, AT=2026-03-31 12:54:00.01
ps_is_running(p)
#> [1] FALSE
px$get_exit_status()
#> [1] -2
```
